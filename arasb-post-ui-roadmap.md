# نقشه راه پس از تایید UI نهایی پروژه ARASB

## 1) وضعیت جدید پروژه
اکنون UI عمومی سایت ARASB مطابق انتظار کارفرما تایید شده است. این یعنی مهم‌ترین ریسک پروژه یعنی عدم تطابق بصری با مرجع اصلی برطرف شده و از این نقطه به بعد باید ظاهر عمومی سایت به‌عنوان مرجع قطعی قفل شود.

اصل اجرایی جدید:

> Exact approved React UI + PHP/MySQL admin/API compatible with IranServer Linux5

یعنی:
- ظاهر عمومی سایت از نسخه تاییدشده React حفظ می‌شود
- بک‌اند، CMS، پنل مدیریت و داده‌های عملیاتی روی مسیر سازگار با هاست Linux5 توسعه پیدا می‌کنند
- هر توسعه جدید باید بدون آسیب به UI فعلی انجام شود

---

## 2) اصل تصمیم‌گیری از اینجا به بعد
از این مرحله به بعد سه اصل غیرقابل‌تغییر داریم:

1. **UI عمومی سایت فریز می‌شود**
   - تغییرات جدید نباید باعث به‌هم‌ریختگی spacing، typography، assets، motion یا responsive behavior شوند.

2. **پنل مدیریت فارسی است**
   - همه بخش‌های ادمین، پیام‌ها، فرم‌ها و تجربه کاربری داخلی به زبان فارسی طراحی و پیاده‌سازی می‌شوند.

3. **سایت عمومی سه‌زبانه است**
   - فارسی
   - انگلیسی
   - عربی

---

## 3) معماری نهایی پیشنهادی

## 3.1 لایه‌ها
### لایه عمومی سایت
- React frontend تاییدشده
- build خروجی استاتیک یا hybrid-compatible برای public layer
- استفاده از API برای محتوای داینامیک

### لایه پنل و API
- PHP 8.1+
- MySQL / MariaDB
- ساختار ماژولار برای admin, api, storage, update system
- سازگار با shared hosting ایران‌سرور

### لایه داده
- پایگاه داده برای موجودیت‌های اصلی
- media storage روی filesystem هاست
- فایل‌های پشتیبان و بسته‌های آپدیت در storage اختصاصی

---

## 3.2 اصل مهم معماری
فرانت عمومی دیگر rebuild نمی‌شود، بلکه **به داده واقعی متصل می‌شود**.

یعنی به‌جای اینکه دوباره UI را در استک دیگری بازسازی کنیم:
- متن‌ها داینامیک می‌شوند
- تصاویر و داده‌ها داینامیک می‌شوند
- فرم‌ها واقعی می‌شوند
- سئو، اسلاگ، ترجمه و محصولات از بک‌اند خوانده می‌شوند

اما لایه UI همان نسخه تاییدشده باقی می‌ماند.

## 4) فازهای اجرایی از اینجا به بعد

## فاز 2 — Freeze + Content Mapping
### هدف
ثبت UI نهایی به‌عنوان baseline و مشخص‌کردن اینکه هر بخش فرانت از چه داده‌ای تغذیه خواهد شد.

### خروجی‌ها
- inventory کامل سکشن‌های صفحه اصلی
- mapping بین componentهای React و داده‌های CMS
- لیست متن‌های قابل ترجمه
- لیست assetهای ثابت و قابل‌مدیریت
- تعیین فیلدهای editable و non-editable

### نتیجه
این فاز مشخص می‌کند هر متن، تصویر، لینک، دکمه، سکشن و meta از کجا در پنل مدیریت شود.
