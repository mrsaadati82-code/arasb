# مسیر اجرایی نهایی پروژه ARASB

## 1) هدف این سند
این سند، مسیر اجرایی واحد و نهایی پروژه ARASB را مشخص می‌کند تا از این مرحله به بعد:
- معماری دوگانه و چندمسیره نداشته باشیم
- تصمیم‌های فنی پراکنده نشوند
- روند کار ثبت شود
- توسعه با یک نقشه راه مشخص جلو برود

---

## 2) وضعیت واقعی فعلی
تا اینجا این موارد مشخص شده‌اند:

### 2.1 آنچه ثابت شده
- هاست ایران‌سرور Linux5 با PHP 8.1 می‌تواند نسخه فعلی prototype را اجرا کند.
- ساختار PHP/Shared Hosting روی هاست عملی است.
- نسخه فعلی public/admin prototype روی هاست بالا آمده است.
- کاربر انتظار دارد ظاهر نهایی **دقیقاً مطابق نسخه مرجع React** باشد.

### 2.2 آنچه هنوز قطعی نیست
- اینکه آیا هاست ایران‌سرور Linux5 اجرای React/Vite/Node-based production build را به شکل مناسب و بدون دردسر پشتیبانی می‌کند یا نه، هنوز به‌صورت عملی و قطعی تایید نشده است.
- داشتن PHP به‌تنهایی به معنی اجرای مستقیم React app به شکل SSR/Node runtime نیست.

---

## 3) تصمیم معماری نهایی فعلاً
### تصمیم اجرایی نهایی فعلی
تا زمانی که خلافش با تست واقعی ثابت نشود، مسیر اصلی پروژه این است:

> **Laravel/PHP-compatible delivery with source-faithful frontend rebuild**

یعنی:
- بستر اجرایی: سازگار با PHP و هاست اشتراکی
- ظاهر فرانت‌اند: بازسازی وفادار به نسخه مرجع React
- پنل مدیریت: فارسی، اختصاصی، سازگار با هاست

---

## 4) چرا این تصمیم گرفته شد؟
1. هاست فعلی عملاً اجرای PHP را تایید کرده است.
2. prototype روی PHP 8.1 بالا آمده است.
3. هدف فعلی، رسیدن به خروجی قابل مشاهده و وفادار به مرجع است.
4. مسیر React/Node hosting روی این پلن هنوز از نظر اجرایی ریسک و ابهام دارد.

---

## 5) آیا React کنار گذاشته شد؟
خیر.

### نقش React در این پروژه از اینجا به بعد
نسخه مرجع React:
- مرجع بصری
- مرجع ساختار سکشن‌ها
- مرجع متن‌ها
- مرجع spacing و motion

است.

اما مسیر اجرایی فعلی این است که همان UI به صورت **port/rebuild دقیق** در بستر سازگار با هاست فعلی بازسازی شود.
