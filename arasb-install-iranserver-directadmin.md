# راهنمای نصب ARASB Prototype روی هاست ایران‌سرور (DirectAdmin)

## 1) هدف این راهنما
این راهنما برای نصب و اجرای نسخه آزمایشی فعلی پروژه ARASB روی هاست لینوکسی اشتراکی ایران‌سرور با کنترل‌پنل DirectAdmin نوشته شده است.

نسخه هدف:
- `arasb-laravel-foundation-v0.9.0.zip`

> این نسخه برای **تست prototype** مناسب است، نه لانچ production نهایی.

---

## 2) پیش‌نیازها
- هاست Linux5 ایران‌سرور
- دسترسی به DirectAdmin
- File Manager یا FTP
- ترجیحاً یک ساب‌دامین تست مثل:
  - `test.yourdomain.com`
  - یا یک پوشه مثل `/dev`

---

## 3) پیشنهاد قبل از نصب
بهتر است این نسخه را روی دامنه اصلی نصب نکنید و ابتدا روی یکی از این حالت‌ها تست کنید:
- ساب‌دامین تست
- پوشه تست داخل دامنه

---

## 4) فایل مورد نیاز
فایل zip مورد استفاده:
- `arasb-laravel-foundation-v0.9.0.zip`

اگر این فایل را از من دریافت کرده‌اید، آن را روی سیستم خود نگه دارید تا در File Manager آپلود کنید.

---

## 5) روش نصب روی ساب‌دامین تست

## گام 1: ساخت ساب‌دامین
در DirectAdmin:
- وارد بخش **Subdomain Management** شوید.
- یک ساب‌دامین مثل `test` بسازید.

مثال:
- `test.yourdomain.com`

پس از ساخت، معمولاً یک مسیر برای آن ساخته می‌شود.

---

## گام 2: ورود به File Manager
- از DirectAdmin وارد **File Manager** شوید.
- وارد مسیر ساب‌دامین تست شوید.

---

## گام 3: آپلود فایل ZIP
- فایل `arasb-laravel-foundation-v0.9.0.zip` را آپلود کنید.

---

## گام 4: Extract فایل ZIP
- روی فایل zip کلیک کنید.
- گزینه **Extract** را بزنید.
- مطمئن شوید فایل‌ها در مسیر ساب‌دامین یا پوشه تست extract شده‌اند.

---

## گام 5: بررسی ساختار فایل‌ها
پس از extract، باید پوشه‌ها و فایل‌هایی شبیه این ببینید:
- `app/`
- `bootstrap/`
- `config/`
- `public/`
- `resources/`
- `storage/`
- `routes/`
- `public/admin/index.php`
- `public/index.php`

---

## 6) نکته مهم درباره public
در ساختار فعلی پروژه، فایل‌های اجرایی داخل پوشه `public` قرار دارند.

در هاست اشتراکی معمولاً باید بررسی کنید که **Document Root** دامنه/ساب‌دامین به کجا اشاره می‌کند.

### حالت ایده‌آل
Document Root ساب‌دامین باید روی پوشه:
- `public/`

قرار بگیرد.

### اگر Document Root روی ریشه پروژه است
ممکن است لازم باشد:
- محتویات پوشه `public` را به ریشه مسیر قابل دسترس وب منتقل کنید
- یا تنظیم ساب‌دامین را طوری انجام دهید که root آن روی `public` باشد

> چون DirectAdmin در سرورهای مختلف ممکن است تنظیم متفاوتی داشته باشد، اگر امکان تعیین document root برای ساب‌دامین دارید، آن را روی `public` بگذارید.

---

## 7) تست آدرس‌های عمومی
بعد از نصب، این URLها را تست کنید:
- `https://test.yourdomain.com/fa`
- `https://test.yourdomain.com/fa/about`
- `https://test.yourdomain.com/fa/products`
- `https://test.yourdomain.com/fa/contact`
- `https://test.yourdomain.com/en`
- `https://test.yourdomain.com/ar`

اگر پروژه را در پوشه نصب کرده‌اید، URLها شبیه این می‌شوند:
- `https://yourdomain.com/dev/fa`

---

## 8) تست پنل مدیریت
برای تست پنل:
- `https://test.yourdomain.com/admin/login`

### ورود آزمایشی
در نسخه فعلی prototype:
- هر ایمیل غیرخالی
- هر رمز غیرخالی

برای ورود اولیه کافی است.

نمونه:
- Email: `admin@arasb.local`
- Password: `123456`

## 9) تست بخش‌های پنل
بعد از ورود، این مسیرها را تست کنید:
- `/admin`
- `/admin/languages`
- `/admin/pages`
- `/admin/pages/edit?id=1`
- `/admin/products`
- `/admin/products/edit?id=1`
- `/admin/settings`

---

## 10) تست ذخیره prototype-level
در این نسخه، ذخیره صفحات و محصولات در فایل‌های JSON انجام می‌شود.

### تست ذخیره صفحه
- وارد `/admin/pages/edit?id=1` شوید
- چند فیلد را تغییر دهید
- ذخیره را بزنید
- باید پیام موفقیت ببینید

### تست ذخیره محصول
- وارد `/admin/products/edit?id=1` شوید
- تغییرات را اعمال کنید
- ذخیره را بزنید
- باید پیام موفقیت ببینید

---

## 11) اگر صفحه لود نشد
### مواردی که باید بررسی کنید
1. آیا فایل‌ها واقعاً extract شده‌اند؟
2. آیا document root درست تنظیم شده؟
3. آیا URL به `public` اشاره درست دارد؟
4. آیا مسیر ساب‌دامین درست است؟
5. آیا فایل‌های CSS لود می‌شوند؟

---

## 12) اگر /admin/login باز نشد
بررسی کنید:
- `public/admin/index.php` در مسیر وب‌قابل‌دسترسی قرار دارد یا نه
- rewrite خاصی لازم نباشد
- ساختار نصب در ریشه وب با پوشه `public` هم‌راستا باشد

---

## 13) نکته امنیتی مهم
این نسخه:
- production-ready نیست
- auth آن آزمایشی است
- برای محیط اصلی و عمومی مناسب نیست

پس حتماً آن را فقط در:
- ساب‌دامین تست
- محیط توسعه
- یا پوشه محدود

اجرا کنید.

---

## 14) جمع‌بندی
برای هاست ایران‌سرور، نسخه `v0.9.0` اکنون برای تست prototype قابل نصب است. مهم‌ترین نکته، تنظیم صحیح مسیر وب روی پوشه `public` یا مدیریت مناسب محتوای آن است.
