# راهنمای استقرار نسخه اصلی React ARASB روی هاست ایران‌سرور

## 1) هدف
این راهنما برای استقرار **همان نسخه اصلی فرانت‌اند React** که مرجع بصری پروژه است نوشته شده است.

فایل آماده تست:
- `arasb-react-static-site.zip`

این فایل از build پروژه اصلی ساخته شده و برای تست UI مرجع مناسب است.

---

## 2) محتوای فایل ZIP
فایل zip شامل این ساختار است:
- `dist/index.html`
- `dist/hero-trading.jpg`
- `dist/saffron.jpg`
- `dist/pistachios.jpg`
- `dist/dates.jpg`
- `dist/petroleum.jpg`

نکته:
- build فعلی به صورت single-file + assets ساخته شده و برای هاست اشتراکی مناسب است.

---

## 3) روش نصب روی ایران‌سرور
### روش پیشنهادی
این build را روی یک ساب‌دامین تست یا مسیر تست نصب کنید.

مثال:
- `test.arasbtrading.com`
- یا `1.arasbtrading.com`

---

## 4) مراحل نصب
### گام 1
وارد DirectAdmin شوید.

### گام 2
به File Manager بروید.

### گام 3
اگر روی ساب‌دامین تست نصب می‌کنید، وارد مسیر root همان ساب‌دامین شوید.

### گام 4
فایل `arasb-react-static-site.zip` را آپلود کنید.

### گام 5
فایل zip را Extract کنید.

### گام 6
محتویات داخل پوشه `dist` را به ریشه قابل دسترس وب منتقل کنید.

یعنی در نهایت باید این فایل‌ها مستقیماً در ریشه وب باشند:
- `index.html`
- `hero-trading.jpg`
- `saffron.jpg`
- `pistachios.jpg`
- `dates.jpg`
- `petroleum.jpg`

نکته خیلی مهم:
- خود پوشه `dist` نباید لایه اضافه بماند، مگر اینکه بخواهی سایت را با `/dist` باز کنی.
- برای تست درست، `index.html` باید در root ساب‌دامین یا مسیر تست باشد.

---

## 5) بعد از نصب چه آدرسی را باز کنم؟
اگر روی ساب‌دامین تست نصب کردی:
- `https://1.arasbtrading.com/`

اگر در پوشه نصب کردی:
- `https://yourdomain.com/test/`

---

## 6) اگر صفحه باز نشد
این موارد را بررسی کن:
1. آیا `index.html` در root قرار گرفته؟
2. آیا فایل‌ها هنوز داخل پوشه `dist` مانده‌اند؟
3. آیا مسیر عکس‌ها کنار `index.html` هستند؟
4. آیا browser cache را خالی کردی؟

---

## 7) این نسخه برای چه چیزی مناسب است؟
- تست UI واقعی مرجع
- بررسی دقیق شباهت با نسخه موردنظر
- گرفتن تایید بصری

## 8) این نسخه برای چه چیزی مناسب نیست؟
- پنل مدیریت
- CMS
- ذخیره محتوا
- فرم‌های backend-driven
- سایت نهایی با مدیریت از پنل

---

## 9) جمع‌بندی
این نسخه بهترین راه برای تست همان UI مرجعی است که کاربر تایید کرده است. بعد از تایید UI، می‌توان برای اتصال backend و CMS تصمیم دقیق‌تری گرفت.
