# تصمیم فنی نهایی اولیه برای هاست Linux5 ایران‌سرور

## 1) مبنا
بر اساس صفحه رسمی هاست لینوکس ایران‌سرور، پلن Linux5 شامل این موارد است:
- 5 گیگ فضای میزبانی
- SSD
- 2 هسته CPU
- 2 گیگ RAM
- کنترل‌پنل DirectAdmin
- 80 گیگ ترافیک ماهانه
- PHP 8.1+
- MySQL
- پشتیبانی مناسب برای PHP/HTML
- وب‌سرور nginx_apache
- SSL رایگان
- بکاپ دیتابیس روزانه و بکاپ کامل هفتگی [1](https://www.iranserver.com/web-hosting/linux/) [2](https://www.iranserver.com/web-hosting/linux/)

همچنین صفحه محصول، هاست لینوکس را به‌عنوان سرویس مناسب برای PHP، MySQL، HTML و CMSهای رایج معرفی می‌کند [1](https://www.iranserver.com/web-hosting/linux/) [2](https://www.iranserver.com/web-hosting/linux/).

---

## 2) نتیجه کلیدی
این پلن، **مبنای خوبی برای اجرای یک سایت PHP/MySQL یا سایت استاتیک** است؛ اما از شواهد صفحه محصول، پشتیبانی شفاف و رسمی برای اجرای معماری مبتنی بر:
- NestJS
- PostgreSQL
- Node.js app server دائمی
- SSR-based Next.js production server

مشخص نیست [1](https://www.iranserver.com/web-hosting/linux/) [2](https://www.iranserver.com/web-hosting/linux/).

بنابراین برای این هاست، معماری‌ای که قبلاً به‌صورت ایده‌آل پیشنهاد شده بود، باید **اصلاح و بومی‌سازی شود**.

---

## 3) تصمیم پیشنهادی برای این هاست
### تصمیم اصلی
برای اینکه پروژه روی Linux5 ایران‌سرور با کمترین ریسک اجرا شود، بهترین مسیر این است:

### سمت سایت عمومی
- خروجی **استاتیک و قابل آپلود**
- فایل‌های build شده به صورت zip تحویل داده شوند
- UI فعلی حفظ شود و صفحات جدید هم با همان ظاهر توسعه یابند

### سمت پنل مدیریت
- اگر بخواهیم حتماً روی همین هاست باشد، بهتر است پنل نیز با **تکنولوژی سازگار با هاست اشتراکی** پیاده‌سازی شود
- یعنی پنل و بک‌اند بهتر است به سمت **PHP + MySQL** بروند، نه NestJS/PostgreSQL

### سمت بک‌اند
- برای این هاست، پیشنهاد قبلی NestJS/PostgreSQL مناسب‌ترین گزینه اجرایی نیست
- گزینه اجرایی منطقی‌تر:
  - **Laravel + MySQL**

## 4) جمع‌بندی تصمیم تکنولوژی جدید
### پیشنهاد اجرایی سازگار با هاست Linux5
- **Frontend Website:** HTML/CSS/JS buildable یا Next export-style static output
- **Admin Panel:** Laravel-based admin
- **Backend:** Laravel
- **Database:** MySQL
- **Media/File storage:** روی همان هاست

### مزایا
- سازگار با هاست ایران‌سرور Linux5
- نصب آسان‌تر روی DirectAdmin
- عدم نیاز به Node server دائمی
- عدم نیاز به PostgreSQL
- قابلیت zip و upload
- پایداری بیشتر روی shared hosting

## 5) اثر این تصمیم روی پروژه
### چیزی که تغییر نمی‌کند
- ظاهر فعلی سایت حفظ می‌شود
- هویت بصری ثابت می‌ماند
- سایت 3 زبانه می‌ماند
- پنل فارسی می‌ماند
- مدیریت صفحات، محصولات، سئو، رسانه، فرم‌ها و آپدیت همچنان جزو نیازمندی‌ها هستند

### چیزی که تغییر می‌کند
- استک بک‌اند از NestJS/PostgreSQL به Laravel/MySQL تغییر می‌کند
- طراحی استقرار بر پایه zip/upload و هاست اشتراکی انجام می‌شود
- معماری فنی باید با محدودیت‌های DirectAdmin و shared hosting تنظیم شود

## 6) پیشنهاد معماری نهایی سازگار با Linux5
### گزینه A — کاملاً سازگار و کم‌ریسک
- سایت + پنل + بک‌اند همگی با Laravel ecosystem
- فرانت عمومی با Blade یا Hybrid rendering
- پنل مدیریت سفارشی فارسی
- MySQL

### گزینه B — ترکیبی
- سایت عمومی به صورت build static
- پنل و API با Laravel
- ارتباط از طریق API یا routeهای داخلی

### پیشنهاد من
برای سرعت، پایداری و سازگاری با هاست شما:
> **گزینه B یا Laravel-centric architecture** بهترین انتخاب است.
