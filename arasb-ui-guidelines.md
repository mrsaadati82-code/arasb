# UI Guidelines پروژه ARASB

## 1) هدف سند
این سند، قواعد اجرایی طراحی UI برای تیم توسعه و طراحی را مشخص می‌کند تا خروجی نهایی هم در سایت و هم در پنل مدیریت، یکپارچه، premium و قابل نگه‌داری باشد.

---

## 2) قواعد کلی سایت عمومی
1. از شلوغی بصری اجتناب شود.
2. هر سکشن باید هدف مشخص داشته باشد.
3. فاصله‌ها generous باشند.
4. CTAها واضح و محدود باشند.
5. تصاویر باکیفیت و مرتبط باشند.
6. انیمیشن‌ها باید subtle باشند.
7. متن‌ها باید ساختارمند و business-grade باشند.

---

## 3) قواعد صفحات جدید
### همه صفحات جدید باید:
- با palette فعلی ساخته شوند
- از container و spacing استاندارد استفاده کنند
- heading hierarchy درست داشته باشند
- CTA مناسب داشته باشند
- mobile-first و responsive باشند
- با RTL/LTR درست کار کنند

---

## 4) قواعد فرم‌ها
- label واضح
- placeholder فقط مکمل، نه جای label
- stateهای خطا/موفقیت واضح
- grouping منطقی فیلدها
- دکمه submit برجسته اما نه خشن
- در پنل، sticky action bar برای فرم‌های طولانی

---

## 5) قواعد جدول‌ها و لیست‌ها در پنل
- ستون‌های ضروری فقط نمایش داده شوند
- actions در دسترس اما تمیز باشند
- badgeهای وضعیت استفاده شود
- فیلترها سریع و واضح باشند
- empty state حرفه‌ای باشد

---

## 6) قواعد سه‌زبانه
- متن فارسی و عربی RTL
- متن انگلیسی LTR
- alignment و spacing برای هر زبان کنترل شود
- typography در هر زبان خوانا بماند
- ترجمه‌های ناقص به وضوح مشخص شوند

---

## 7) قواعد پنل مدیریت
- هر صفحه یک عنوان واضح داشته باشد
- description کوتاه برای صفحات پیچیده باشد
- مسیر breadcrumb مشخص باشد
- عملیات اصلی بالا و در دسترس باشند
- فرم‌های پیچیده tabbed یا sectioned باشند
- بازخوردهای سیستم فوری و واضح باشند

---

## 8) قواعد حفظ کیفیت خروجی
- component reuse اجباری
- ساخت دستی و پراکنده ممنوع
- قبل از ساخت هر صفحه جدید، باید از الگوهای موجود استفاده شود
- هر UI جدید باید با design tokens هماهنگ باشد

---

## 9) جمع‌بندی
این guidelines جلوی واگرایی طراحی را می‌گیرد و کمک می‌کند ظاهر فعلی ARASB حفظ و حرفه‌ای‌تر شود، نه اینکه با توسعه صفحات جدید تضعیف شود.
